Resumen de XML

Tema 10: Definición, esquema y vocabulario

¿Qué es XML?

XML (Extensible Markup Language) es un lenguaje de marcado definido por el W3C. Su propósito principal no es mostrar datos, sino describir datos y compartirlos a través de diferentes sistemas.

  • Es un lenguaje de estructura libre donde las etiquetas no están predefinidas.
  • No es un sustituto de HTML, ni un lenguaje de programación.

Documento Bien Formado

Para que un documento pueda procesarse correctamente, debe estar bien formado cumpliendo reglas sintácticas:

  • Debe tener una estructura de árbol con un único nodo raíz.
  • Las etiquetas deben estar emparejadas (apertura y cierre).
  • Los atributos deben ir siempre entre comillas.
Si un XML no está bien formado dará error al instante.

Validación (DTD)

Validar un XML es comprobar que cumple con las reglas de estructura y contenido predefinidas mediante un DTD (Document Type Definition).

  • Un DTD describe la estructura del documento.
  • Define qué elementos y atributos pueden existir usando las palabras clave ELEMENT y ATTLIST.

El Parser

Es la aplicación que lee un XML y lo convierte en una estructura que el ordenador puede usar, permitiendo detectar errores y trabajar con los datos.

Formatos de intercambio

Existen alternativas populares a XML para intercambiar información:

  • JSON: Ligero y basado en pares clave-valor.
  • YAML: Usa indentación y es muy fácil de leer.
  • CSV: Formato tabular delimitado por comas.
