Tema 12: Conversión y Adaptación XML

Un path es una ruta o camino hacia algo. XPath es un lenguaje que permite evaluar expresiones, localizar elementos, extraer datos y hacer cálculos dentro del árbol de un documento XML.

Lenguajes de Transformación XSL

XSL es una ampliación de CSS. A diferencia de CSS, donde la transformación es puramente visual, XSL incluye:

  • XPath: Evalúa expresiones y localiza elementos.
  • XSLT: Transforma el documento en HTML u otro XML.
  • XSL-FO: Transforma a otros formatos como PDF.

El Árbol del Documento XML

Los documentos XML se estructuran por nodos (ascendientes, descendientes, hermanos). Tipos principales:

  • Raíz: El nodo raíz siempre es /.
  • Elemento: Representan elementos del documento.
  • Atributo: Nodos de atributos.
  • Texto: El contenido de texto.

Sintaxis y Expresiones XPath

Una expresión XPath es una cadena de texto que localiza nodos de un árbol XML. Sus partes principales son:

  • Eje: Seleccionar elementos o atributos.
  • Predicado: Aplicando condiciones al eje.
  • Selección: Indica resultado final.

Formatos de Intercambio de Datos

Formato Características Principales Usos Comunes
XML Basado en etiquetas. Jerárquico y extensible. Verboso, pero flexible. Integración de sistemas, documentos jerárquicos.
JSON Ligero y legible. Basado en pares clave-valor. APIs REST, Eficiente en aplicaciones web.
CSV Tabular y simple. Delimitado por comas o punto y coma. Datos simples, Importación/exportación.
YAML Usa indentación para anidación. Legible por humanos. Archivos de configuración (ej. Kubernetes).
